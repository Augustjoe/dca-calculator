# 定投计算器需求文档

## 1. 项目概述

定投计算器是一个纯前端历史回测工具，用于测算预置股票、ETF 和指数在指定历史区间内的单笔买入收益或定期定额投资收益。

项目当前按静态站点交付，不提供后端服务、不提供用户登录、不在浏览器端实时抓取任意标的数据。行情数据通过本地脚本更新为静态 JSON，并随前端一起部署到 Netlify。

当前线上地址：

- <https://augustjoe-dca-calculator.netlify.app>

## 2. 当前版本目标

1. 支持用户从预置标的中选择股票、ETF 或指数。
2. 支持单笔买入收益测算。
3. 支持每日、每周、每月定投计划回测。
4. 基于历史前复权收盘价计算收益。
5. 展示投入金额、期末资产、收益金额、收益率、简化年化收益率、买入份额等核心指标。
6. 展示资产价值曲线，并支持全屏查看。
7. 单笔买入默认只展示历史持仓价值；最大回撤作为可选辅助线保留。
8. 定投计划展示资产市值与标的走势。
9. 切换计算模式时保留当前表单输入，并避免结果区域闪烁为空状态。
10. 适配桌面三栏布局与移动端上下堆叠布局。

## 5. 数据方案

### 5.1 数据来源

数据更新脚本位于 `scripts/update-data.py`。

当前数据来源策略：

- A 股与 ETF：优先尝试 AKShare，网络不可用时使用兜底数据源。
- 美股指数：使用 Yahoo Finance。
- 中证指数：使用 CSIndex/AKShare 相关接口。

数据采用前复权口径，静态输出只保留回测必需字段。

### 5.2 数据文件结构

```text
public/
  data/
    manifest.json
    stocks/
      600519.SH.json
      NDX.US.json
      000300.CSI.json
```

`public/data/manifest.json` 结构：

```json
{
  "updatedAt": "2026-06-08T22:43:50+08:00",
  "stocks": [
    {
      "symbol": "600519",
      "market": "SH",
      "name": "贵州茅台",
      "assetType": "stock",
      "file": "/data/stocks/600519.SH.json",
      "latestTradeDate": "2026-06-08"
    }
  ]
}
```

单个行情文件结构：

```json
{
  "symbol": "600519",
  "market": "SH",
  "name": "贵州茅台",
  "assetType": "stock",
  "adjust": "qfq",
  "source": "Yahoo Finance",
  "updatedAt": "2026-06-08T22:43:50+08:00",
  "latestTradeDate": "2026-06-08",
  "prices": [["2020-01-02", 979.01]]
}
```

### 5.3 数据更新流程

```bash
npm run update-data
npm run test
npm run build
```

更新完成后，需将 `public/data/manifest.json` 与 `public/data/stocks/*.json` 一起提交到 GitHub，再部署到 Netlify。

- `2026-06-08T22:43:50+08:00`

## 6. 计算模式

### 6.1 单笔买入

输入项：

- 标的
- 买入日
- 查看日
- 买入金额

交易日规则：

- 买入日如果不是交易日，使用该日期之后第一个交易日。
- 查看日如果不是交易日，使用该日期之前或当天最后一个交易日。

计算规则：

```text
买入份额 = 买入金额 / 买入日收盘价
期末资产 = 买入份额 * 查看日收盘价
收益金额 = 期末资产 - 买入金额
收益率 = 收益金额 / 买入金额
```

图表：

- 默认展示历史持仓价值。
- 保留“最大回撤”按钮，点击后显示截至每个日期的历史最大回撤辅助线。
- 支持全屏查看走势图，全屏视图支持缩放条。

### 6.2 定投计划

输入项：

- 标的
- 开始日
- 结束日
- 每次定投金额
- 定投频率：每日、每周、每月

交易日规则：

- 每日定投只在交易日买入。
- 每周定投使用每个自然周内第一个交易日。
- 每月定投使用每个自然月内第一个交易日。
- 结束日如果不是交易日，使用该日期之前或当天最后一个交易日。

计算规则：

```text
每次买入份额 = 每次定投金额 / 当日收盘价
累计份额 += 每次买入份额
累计投入 += 每次定投金额
期末资产 = 累计份额 * 结束日收盘价
收益金额 = 期末资产 - 累计投入
收益率 = 收益金额 / 累计投入
```

图表：

- 展示资产市值。
- 展示标的走势，使用右侧 Y 轴。
- 支持全屏查看走势图，全屏视图支持缩放条。

## 7. 收益指标

当前展示：

- 总投入 / 买入金额
- 当前价值 / 期末资产
- 收益金额
- 收益率
- 简化年化收益率
- 买入价格
- 累计份额 / 买入份额
- 买入次数（定投模式）

简化年化公式：

```text
简化年化收益率 = (期末资产 / 总投入) ^ (1 / 投资年数) - 1
```

注意：

- 单笔买入使用普通年化收益率较合理。
- 定投计划存在持续现金流，当前简化年化只作为粗略参考。
- 后续建议增加 XIRR，更适合定投现金流场景。

## 8. UI 与交互

当前 UI 使用 Ant Design。

桌面端布局：

- 左侧：收益测算表单
- 中间：价值变化与图表
- 右侧：收益摘要与计算规则

移动端布局：

- 上下堆叠
- 保持表单、图表、摘要可读
- 避免横向滚动和文字重叠

交互要求：

- 计算模式切换时不重置表单。
- 计算模式切换时不先清空结果区域，避免闪烁。
- 数据加载失败、日期越界、结束日期早于开始日期、金额无效、区间内无交易日时，页面内展示错误提示。
- 最大回撤功能默认隐藏，仅用户点击按钮后展示。

## 11. 当前非目标

当前版本不做：

1. 用户登录。
2. 用户自定义任意标的实时查询。
3. 后端服务。
4. 数据库。
5. 实时行情。
6. 全市场数据缓存。
7. 真实手续费、印花税、过户费。
8. 买卖建议。
9. 投资组合回测。
10. 买入明细表与 CSV 导出。
11. XIRR。
