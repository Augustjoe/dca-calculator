# A 股定投收益回测工具需求文档

## 1. 项目概述

本项目是一个轻量级纯前端网页工具，用于测算指定 A 股股票在某个历史时间段内的单笔买入收益或定期定额投资收益。

第一版不做全市场实时查询，也不做后端服务。项目部署前在本地维护一份 `watchlist`，最多选择 10 支股票。运行数据更新脚本后，将这 10 支股票的历史行情保存为静态 JSON 文件，并随前端项目一起打包部署到 Netlify。

上线后，用户只能查询这 10 支预置股票的数据。

## 2. 核心目标

1. 支持用户选择一只预置 A 股股票。
2. 支持选择投资开始日期和结束日期。
3. 支持单笔买入收益测算：用户输入买入日期、买入金额和查看日期，系统展示期末价值、收益和收益率。
4. 支持定投收益回测：用户输入日期区间、每次定投金额和定投频率，系统展示累计投入、期末资产、收益和收益率。
5. 基于历史前复权收盘价计算收益。
6. 展示总投入、期末资产、收益金额、收益率、年化收益率等结果。
7. 展示资产变化曲线。
8. 部署方式尽量简单，第一版使用 Netlify 静态部署。

## 3. 第一版范围

### 3.1 支持的股票范围

第一版只支持部署前预置的最多 10 支 A 股股票。

股票列表由本地文件维护，例如：

```json
[
  { "symbol": "000001", "market": "SZ", "name": "平安银行" },
  { "symbol": "600519", "market": "SH", "name": "贵州茅台" },
  { "symbol": "510300", "market": "SH", "name": "沪深300ETF" }
]
```

前端页面不提供任意股票代码实时查询。用户只能从预置股票列表中选择。

### 3.2 投资方式

第一版支持两个相互独立的计算模式：

1. 单笔买入
   - 用户输入买入日期。
   - 用户输入买入金额。
   - 用户输入查看日期。
   - 系统按买入日期附近的有效交易日收盘价买入。
   - 系统按查看日期附近的有效交易日收盘价计算期末价值。
   - 展示买入金额、期末价值、收益金额、收益率，并展示这笔资金在买入日至查看日期之间的价值变化曲线。

2. 定投计划
   - 用户输入开始日期和结束日期。
   - 用户输入每次定投金额。
   - 用户选择定投频率。
   - 第一版建议支持：
     - 每日
     - 每周
     - 每月
   - 展示累计投入、期末资产、收益金额、收益率，并展示资产价值和累计投入在区间内的变化曲线。

第一版暂不把单笔买入和定投混合成同一个策略。用户每次计算时选择其中一种模式。

### 3.3 计算规则

第一版采用理论回测模式：

1. 使用前复权收盘价。
2. 允许碎股。
3. 暂不考虑手续费。
4. 暂不考虑印花税。
5. 暂不单独展示分红现金流。
6. 周末和节假日不交易。
7. 单笔买入日期若非交易日，则使用该日期之后的第一个可用交易日。
8. 查看日期或结束日期若非交易日，则使用该日期之前或当天的最后一个可用交易日。
9. 定投日若非交易日，则顺延到下一个可用交易日，或使用该周期内第一个可用交易日。

后续版本可以增加真实 A 股交易模式：

1. 按 100 股整数倍买入。
2. 未使用资金作为现金留存。
3. 加入佣金、印花税、过户费等交易成本。
4. 基于未复权价格和分红送转数据做更精确模拟。

## 4. 数据方案

### 4.1 数据来源

第一版建议使用 AKShare 在本地拉取 A 股历史行情。

使用前复权日线数据，字段只保留回测必要信息。

建议数据字段：

```json
{
  "symbol": "600519",
  "market": "SH",
  "name": "贵州茅台",
  "adjust": "qfq",
  "source": "AKShare",
  "updatedAt": "2026-06-04T17:00:00+08:00",
  "latestTradeDate": "2026-06-04",
  "prices": [
    ["2020-01-02", 1001.23],
    ["2020-01-03", 1008.5]
  ]
}
```

### 4.2 数据文件结构

建议项目结构：

```text
a-share-dca-calculator/
  public/
    data/
      manifest.json
      stocks/
        000001.SZ.json
        600519.SH.json
        510300.SH.json
  scripts/
    update-data.py
  src/
    ...
  watchlist.json
  package.json
  REQUIREMENTS.md
```

`manifest.json` 用于告诉前端有哪些股票可用：

```json
{
  "updatedAt": "2026-06-04T17:00:00+08:00",
  "stocks": [
    {
      "symbol": "600519",
      "market": "SH",
      "name": "贵州茅台",
      "file": "/data/stocks/600519.SH.json",
      "latestTradeDate": "2026-06-04"
    }
  ]
}
```

### 4.3 数据更新流程

本地维护 `watchlist.json`。

当需要更新股票数据时，运行：

```bash
npm run update-data
```

该命令执行数据脚本：

```text
读取 watchlist.json
  ↓
逐个拉取最多 10 支股票的前复权日线数据
  ↓
生成 public/data/stocks/*.json
  ↓
生成 public/data/manifest.json
```

更新完成后，将数据文件和代码一起提交到 GitHub，再由 Netlify 自动部署。

### 4.4 数据量控制

第一版最多 10 支股票，每支股票只保存日期和前复权收盘价。

这样即使保存 10 年以上日线数据，总体积也较小，适合静态部署。

不做以下事情：

1. 不预下载全 A 股。
2. 不保存分钟线。
3. 不保存完整 OHLCV，除非后续确实需要。
4. 不在浏览器端实时抓取任意股票数据。

## 5. 前端功能需求

### 5.1 输入项

页面需要提供：

1. 股票选择
   - 下拉选择预置股票。
   - 显示股票代码、名称、数据截止日期。

2. 日期范围
   - 开始日期。
   - 结束日期。
   - 日期范围必须落在该股票已有数据范围内。

3. 计算模式
   - 单笔买入。
   - 定投计划。

4. 单笔买入输入项
   - 买入日期。
   - 查看日期。
   - 买入金额。

5. 定投计划输入项
   - 开始日期。
   - 结束日期。
   - 每次定投金额。
   - 定投频率：每日、每周、每月。

6. 计算按钮
   - 点击后执行回测。

### 5.2 输出项

计算后展示：

1. 总投入金额。
2. 期末资产。
3. 总收益金额。
4. 总收益率。
5. 年化收益率。
6. 累计买入份额。
7. 数据截止日期。
8. 计算规则。

建议后续增加：

1. 最大回撤。
2. 投入本金曲线。
3. 资产市值曲线。
4. 每次买入明细表。

### 5.3 图表

第一版至少展示一张资产曲线图。

建议图表包含：

1. 资产市值曲线。
2. 累计投入曲线。
3. 明确坐标轴含义：X 轴表示时间，Y 轴表示价值（元）。

可选图表：

1. 股票价格曲线。
2. 收益率曲线。
3. 买入点标记。

## 6. 计算逻辑

### 6.1 单笔买入

```text
买入份额 = 买入金额 / 买入日价格
期末资产 = 当前总份额 * 结束日价格
收益 = 期末资产 - 买入金额
收益率 = 收益 / 买入金额
```

单笔买入模式下，资产曲线按区间内每个交易日计算：

```text
每日资产价值 = 买入份额 * 当日价格
```

### 6.2 定投计划

```text
每个定投日：
  买入份额 = 定投金额 / 当日价格
  累计份额 += 买入份额
  累计投入 += 定投金额

期末资产 = 累计份额 * 结束日价格
收益 = 期末资产 - 累计投入
收益率 = 收益 / 累计投入
```

定投计划模式下，资产曲线按区间内每个交易日计算：

```text
每日资产价值 = 当前累计份额 * 当日价格
每日累计投入 = 截至当日已投入金额
```

### 6.3 年化收益率

```text
年化收益率 = (期末资产 / 总投入) ^ (1 / 投资年数) - 1
```

注意：单笔买入可以使用普通年化收益率。定投计划存在持续现金流，普通年化收益率只是简化口径；后续版本可以增加 XIRR，更适合多次现金流场景。

### 6.4 交易日处理

如果用户选择的买入日期、开始日期、查看日期或结束日期不是交易日：

1. 买入日期和开始日期使用该日期之后的第一个可用交易日。
2. 查看日期和结束日期使用该日期之前或当天的最后一个可用交易日。

如果定投日不是交易日：

1. 每日定投：只在交易日买入。
2. 每周定投：选择该周第一个可用交易日，或约定星期一附近的下一个可用交易日。
3. 每月定投：选择该月第一个可用交易日，或用户指定日期附近的下一个可用交易日。

第一版建议先采用简单规则：每周和每月使用周期内第一个交易日。

## 7. 技术方案

### 7.1 推荐技术栈

前端：

```text
Vite + React
```

图表：

```text
ECharts 或 Recharts
```

数据更新脚本：

```text
Python + AKShare
```

部署：

```text
Netlify 静态部署
```

### 7.2 package.json 脚本

建议提供：

```json
{
  "scripts": {
    "dev": "vite",
    "update-data": "python scripts/update-data.py",
    "build": "vite build",
    "preview": "vite preview"
  }
}
```

是否在 `prebuild` 自动更新数据可以后续决定。第一版建议手动运行 `npm run update-data`，避免每次构建都依赖外部数据源。

## 8. 部署方案

第一版部署流程：

```text
本地修改 watchlist.json
  ↓
npm run update-data
  ↓
检查 public/data/manifest.json 和股票 JSON
  ↓
npm run build
  ↓
提交到 GitHub
  ↓
Netlify 自动部署
```

Netlify 只负责部署静态前端，不运行后端服务。

## 9. 非目标

第一版不做：

1. 用户登录。
2. 用户自定义任意股票。
3. 实时行情。
4. 全 A 股数据缓存。
5. 后端服务。
6. 数据库。
7. 真实手续费和税费。
8. 股票买卖建议。
9. 单笔买入和定投混合策略。
10. 投资组合回测。
11. 港股、美股、基金数据。

## 10. 风险和注意事项

1. AKShare 等免费数据源适合个人研究和原型验证，公开商业化前需要重新确认数据授权。
2. 前复权价格适合做总回报近似回测，但不等于真实成交价格。
3. 持续定投场景下，普通年化收益率不如 XIRR 精确。
4. 静态数据不会自动更新，需要本地运行脚本并重新部署。
5. 网站需要明确标注：历史回测仅供参考，不构成投资建议。

## 11. 后续扩展方向

3. 增加手续费、印花税、过户费。

6. 支持 ETF 和基金净值。

1. 初始化 Vite + React 项目。

6. 实现基础回测计算。
